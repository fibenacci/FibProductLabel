# Security Policy

## Reporting a Vulnerability

I take the security of this plugin seriously. If you discover a security vulnerability within this project, please report it privately.

**Do not open a public issue.**

Please send your report via email to:
**benjamin.letzel@protonmail.com**

### What to include in a report?
- A detailed description of the vulnerability.
- Steps to reproduce the issue (PoC).
- Potential impact.

### Response Process
After receiving a report, I will:
1. Acknowledge receipt within 48 hours.
2. Work on a fix and validate it.
3. Provide a timeline for a security release.
4. Credit the reporter (optional, upon request).

## Supported Versions

Only the latest version of this plugin receives security updates. I recommend always staying on the latest release.
